# 지금 한 번만 배포하면 되는 것

**배포하라는 것** = **toeic-words** 이 **폴더 전체**를 Vercel에 다시 올리는 거예요.  
파일 하나씩 고를 필요 없어요. **폴더 통째로** 올리면 됩니다.

(우리가 수정한 건 그 안의 **api/notion-words.js** 한 개뿐이에요. 그래도 배포할 때는 **toeic-words 폴더 전체**를 올려야 해요.)

---

## 방법 1: GitHub 쓰는 경우 (이미 toeic-words를 GitHub에 올려 둔 경우)

1. **toeic-words** 폴더에서 수정한 걸 저장했는지 확인 (api/notion-words.js).
2. 터미널에서:
   ```bash
   cd "c:\Users\admin\OneDrive\바탕 화면\똑패스\toeic-words"
   git add .
   git commit -m "노션 DB 컬럼 Name/제목 인식"
   git push
   ```
3. **Vercel**이 GitHub랑 연결돼 있으면, 푸시만 하면 **자동으로 다시 배포**돼요.  
   Vercel 대시보드 → Deployments 에서 맨 위에 새 배포가 생기면 끝.

---

## 방법 2: Vercel CLI로 올리는 경우 (Git 안 쓰는 경우)

1. 터미널에서:
   ```bash
   cd "c:\Users\admin\OneDrive\바탕 화면\똑패스\toeic-words"
   vercel --prod
   ```
2. 이미 연결된 프로젝트라고 나오면 **Y** 하고 엔터.  
   끝나면 배포된 주소가 나옵니다.

(Vercel CLI가 없으면 `npm i -g vercel` 한 번 설치한 뒤 위 명령 실행.)

---

## 정리

| 질문 | 답 |
|------|----|
| 뭘 배포하라고? | **toeic-words** 이 프로젝트(폴더) 전체 |
| 파일 많아서 뭘 올려? | **toeic-words 폴더 전체** 한 번에 올리면 됨 (파일 골라서 올리기 X) |
| 어떻게 올려? | GitHub 쓰면 → **toeic-words에서 git push** / 안 쓰면 → **toeic-words에서 vercel --prod** |

배포 끝나면 아까 알려드린 **API 주소** 브라우저에서 다시 열어보면 됩니다.
